# TripCart — Design Context

> 정본은 [`docs/DESIGN_SYSTEM.md`](docs/DESIGN_SYSTEM.md)와 [`docs/PRODUCT_MASTER_PLAN.md`](docs/PRODUCT_MASTER_PLAN.md). 본 파일은 `/impeccable` 스킬을 위한 압축 브리프(projection).

## Design Context

### Users
- **1차**: 국내여행(부산 MVP)을 자주 다니는 20-30대. 맛집·카페·관광지 저장 습관이 있고, 당일치기 / 1박2일 코스를 **직접 짜는** 사람.
- **2차**: 커플·친구 단위 자동차 이동, 공유받은 코스를 그대로 따라가는 사용자.
- **사용 맥락 2종**:
  - **공개면 (`/`, `/places`, `/saved-places`, `/plans`)**: 여행 전 집/카페에서 탐색·담기·초안 작성. 비로그인 게스트 상태로도 써볼 수 있어야 함. 감성·신뢰 전달이 우선.
  - **운영면 (`/plans/[id]`, execution)**: 여행 중 이동 차내/현장. 영업시간·브레이크타임·이동시간 확인. 정보 밀도와 빠른 판단이 우선.
- **Job to be done**: "담아 둔 장소들을 영업시간·BT·이동시간까지 반영해 **실행 가능한 하루 일정**으로 정렬하고, 실행 중에는 놓치지 않게 한다."

### Brand Personality
- **3-word tone**: `trustworthy · sharp · travel-warm` (믿을 수 있는 · 또렷한 · 여행 감성)
- **Voice**: 과장 없이 또렷하게. "이 시간엔 BT, 이 구간은 15분" 같이 **숫자·제약**을 담담히 말하는 운영자 톤.
- **Emotional goals**: 공개면에선 **설렘 + 신뢰**, 운영면에선 **안정 + 확실함**. 두 모드가 같은 시각 언어 위에서 전환돼야 한다.

### Aesthetic Direction
- **Theme**: Light. 기본 배경은 밝고 부드러운 wash (해안·하늘·모래의 희미한 haze). 운영면은 같은 light이되 데이터 밀도 우선.
- **References (good)**: Apple App Store 에디토리얼 카드, Airbnb 장소 상세 (장식보다 사진·레이아웃), 일본 여행 가이드북의 편집 그리드, 한국 관광공사 부산 캠페인 페이지 톤.
- **Anti-references (bad)**:
  - 개발자용 링크 허브 (기존 `/`의 문제였음)
  - 피드형 무한 콘텐츠 리스트 (우리는 **담기 → 정렬**이 메타포)
  - AI slop fingerprints — gradient text, cyan-on-dark, side-stripe `border-left`, 아이콘+제목이 반복되는 identical card grid, glassmorphism 과용, neon on dark
  - 전통 "관광 앱" 포스터 같은 과장된 hero copy
- **Color**: mint/teal 베이스 (`primary-500 #2A9D8F`). Plum(`#641A41`)은 **공유·장바구니 count** 한정. Coral/Gold는 **상태 경고 전용**(BT 위험·마감 임박). 공개면 hero 사진 위 frosted haze OK.
- **Typography**:
  - **Body/UI**: Pretendard (한국어 가독성 + Latin proportional weight). 이미 프로젝트 기본.
  - **Display (제목)**: Pretendard 900/800 vs 현재 400/700 중심 — display-weight로 위계를 더 벌려야 함. 또는 한글 editorial 계열 display 폰트 도입 검토 (예: Gmarket Sans, Paperlogy). *확정 전 후보 단계.*
  - **Number/Mono**: JetBrains Mono — 시간·거리·금액·ETA는 항상 mono로. (DESIGN_SYSTEM 3.2 `number-md/lg` 토큰)
- **Layout**:
  - 공개면: 히어로 사진 + 넓은 여백 + 편집형 카드 rail
  - 운영면: 좌 타임라인 / 중 대표 카드 / 우 경로 요약 (3단), 수치는 mono 컬럼으로 정렬
  - 모바일: 지도 + bottom sheet (peek/half/full snap)
- **Motion**: 페이지 전환보다 **상태 변화**에 집중 (save/remove optimistic, reorder, BT 경고 페이드 등). 스태거드 카드 reveal은 공개면 허용, 운영면은 zero-motion 원칙.

### Design Principles (5 loadstars)
1. **Plan과 Execution은 섞지 않는다.** 공개면은 감성·전환, 운영면은 밀도·실행. 같은 토큰 위에서 다른 모드를 입는다.
2. **상태가 가장 먼저 보인다.** 영업중 / BT 위험 / 마감 임박 / 고정됨 — 아이콘 + 텍스트로 병기. 색만으로 구분 금지.
3. **Mono는 숫자, Sans는 문장.** 시간·거리·금액·ETA·요일 전부 mono. 본문과 타이틀은 Pretendard. 숫자 컬럼 정렬로 읽기 속도를 올린다.
4. **Mint 10% · Plum 3% · 나머지 neutral wash.** 브랜드 색은 CTA·링크·경로선에만. Plum은 공유·count 전용. 지도 위에서 브랜드색은 마커와 경로선만.
5. **장식보다 정보, 정보보다 흐름.** 히어로 사진·감성은 공개면 입구에 한정. 본문에선 카드를 덜 겹치고(card-in-card 금지), 여백 리듬으로 위계를 만든다. Modal은 최후 수단.

### Constraints (technical)
- Next.js 16 (App Router) + Tailwind v4 `@theme`, CSS variables는 `packages/design-tokens/tokens.css`.
- React 19.2, RSC 경계 존중 (현재 랜딩/플랜 등은 `'use client'` 중심 — 필요시 분할).
- Expo SDK 54 모바일 — React Native, `StyleSheet` + 공유 `design-tokens`.
- WCAG AA 이상, touch target ≥ 44×44, 상태는 색+텍스트 병기, `prefers-reduced-motion` 존중.
- CI 게이트: `pnpm lint · typecheck · test · build` + `design-quality.test.ts` guardrail (ad-hoc hex 금지, oversized radii 금지, API 보기 노출 금지, 정적 accessibilityLabel 필수).

### Known Pain Points (going into redesign)
- 공개 `/` 히어로 타이포 hierarchy가 평평 — `TRIP CART` 하나만 크고, 서브 카피가 작은 체급. 설렘·신뢰가 약함.
- `/places` split view의 좌측 리스트 카드와 우측 "지도 일러스트"가 서로 다른 언어. 지도 일러스트는 placeholder 티가 나서 신뢰 하락.
- `/plans` 대시보드 카드 3장이 **똑같은 비율/똑같은 뱃지 위치**로 반복 — "identical card grid" AI slop.
- `/plans/[id]` 3단 레이아웃의 우측 "MapPreview"가 SVG placeholder — 정보성도, 감성도 애매.
- 공개면 radial-gradient wash가 3개 페이지에서 거의 동일 — 화면 구분 약함.
- 상태 배지가 `bg-primary-50 text-primary-700` 하나로 통일돼 있어 "예정 / 초안 / 진행중" 차이가 시각적으로 구별되지 않음.
